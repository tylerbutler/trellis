title: Overview
description: What trellis is, how it derives its workspace model from gleam.toml, and where to start.

Trellis is one binary that provides task fan-out, dependency-graph
introspection, changelogs, versioning, and publish orchestration for
multi-package Gleam repositories — all derived from the `gleam.toml` files
you already have.

Trellis derives what it can, and verifies what must remain explicit. The
member set, its topological order, change impact, and publish order all come
from manifests that already exist; the things that can't be derived — locked
versions, changelogs, tags, toolchain pins — are checked by
`trellis doctor`, which exits non-zero when an invariant breaks.

## Getting started

1. [Install trellis](/docs/installation/) — a single prebuilt binary; shell
   installer, Homebrew, mise/asdf, or cargo.
2. [Configure the workspace](/docs/configuration/) — one `[tools.trellis]`
   table in `gleam.toml`; only `members` is required.
3. Run `trellis doctor` to verify the setup, then
   [run tasks](/docs/task-running/) across the workspace.
